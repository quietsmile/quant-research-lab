# 学习路线图与通关标准

本路线图把 `todo.txt` 的阶段 1 细化为可验收的 Gate，并补出后续阶段（todo.txt 未覆盖）。

## 阶段 1 ｜ 技能与工具搭建（2–3 个月）—— 本框架覆盖

**能力点 → 框架落点：**

- 量化研究方法论 → `quantlab.validation`（train/test、walk-forward、过拟合诊断）
- Python 数据栈 → `quantlab.data`（加载、清洗、缓存）
- 统计基础 → `quantlab.stats`（收益/波动/夏普/最大回撤/相关/回归）
- 交易成本与 A 股摩擦 → `quantlab.costs.AShareCostModel`
- 回测 → `quantlab.backtest.Backtester`
- 玩具策略 → `quantlab.strategies.MACrossStrategy`

**Gate 1 通关标准（自测清单）：**

- [ ] 能独立把一个策略从数据跑到带成本的回测（`python examples/toy_strategy.py` 跑通）
- [ ] 能说清回测里每一项成本的来源与量级
- [ ] 能用 walk-forward 给出样本外指标，并解释样本内/外差距
- [ ] 能指出手上这个策略**为什么可能是过拟合**（参数搜索空间、样本量、单一标的等）

## 阶段 2 ｜ 因子与组合（规划中）

- 横截面因子（价值/动量/质量），IC/IR、分层回测
- 组合构建：等权/风险平价/均值方差，约束与换手控制
- Gate 2：多因子组合样本外 IR 稳定为正，且能解释每个因子的经济学逻辑

## 阶段 3 ｜ 风险与执行（规划中）

- 风险模型、回撤控制、仓位管理
- 执行：撮合细节、冲击成本、容量分析
- Gate 3：在容量约束下策略仍有效，回撤可控

## 阶段 4 ｜ 实盘与迭代（规划中）

- 模拟盘 → 小资金实盘，监控与归因
- 研究日志、版本管理、策略退役机制
- Gate 4：实盘与回测偏差在可解释范围内
