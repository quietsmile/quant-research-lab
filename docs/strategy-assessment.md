# 对 `todo.txt` 学习策略的评审

> 结论先行：**todo.txt 的「阶段 1」是一份高于平均水平的学习路线**——它把绝大多数初学者会忽略、却恰恰决定成败的"研究纪律"放在了第一位。下面是逐条评审与改进建议，本框架即是对这份路线的代码化落地。

## 一、它做对了什么（值得保留）

1. **把"量化研究方法论"排在最重要、并明说它最容易被跳过。**
   这是这份 todo 最有价值的判断。过拟合、多重检验、样本内/外、walk-forward、前视偏差、幸存者偏差——初学者 90% 的"赚钱策略"死在这里。先建立"不骗自己"的能力，再谈收益，方向完全正确。

2. **强调把真实摩擦写进回测**（印花税、佣金、滑点、T+1、涨跌停）。
   A 股与美股最大的差异就在交易摩擦与制度约束。很多看起来年化 50% 的策略，扣完 T+1 和涨跌停撮合假设后归零甚至为负。把它列为核心，专业。

3. **端到端"玩具策略"作为交付物**，而不是停留在看课。
   "数据 → 处理 → 信号 → 带成本回测"跑通一遍，胜过看十门课。以"能独立跑通 + 能说清为什么可能过拟合"作为 Gate，标准设得很实。

4. **工具选型务实**：免费数据（AKShare/Tushare）起步、成熟回测框架、把 AI 当生产力工具。没有一上来就上昂贵数据或自研引擎，符合学习阶段的成本/收益。

## 二、可以改进的地方（本框架已据此调整）

| todo.txt 的不足 | 风险 | 本框架的应对 |
|---|---|---|
| 把方法论列为"要学"，但**没有给出可执行的检查动作** | 知道概念 ≠ 做得到，容易学完照样过拟合 | `quantlab.validation` 把 train/test、walk-forward、过拟合诊断做成**函数**，回测默认产出样本外结果 |
| 成本建模"写进回测"是目标，但**没有量化默认值** | 每人参数不一，结果不可比 | `quantlab.costs.AShareCostModel` 给出 A 股**可引用的默认参数**并可覆盖 |
| 工具依赖外部数据源 | 没网/没 token 就**学不下去** | 内置**可复现合成行情**，框架开箱即跑，数据源只是适配器 |
| 缺少"指标定义"的统一口径 | 夏普/回撤算法不一致导致自欺 | `quantlab.stats` 给出**单一可信实现**并有单测 |
| 只有阶段 1 | 看不到全貌、容易半途 | `docs/roadmap.md` 补出后续阶段与各自 Gate |

## 三、给学习者的三条提醒

1. **先怀疑，再相信。** 任何一个回测结果出来，第一反应应是"它哪里可能骗了我"，而不是"我能投多少钱"。本框架的 `summary()` 会主动列出可疑点。
2. **样本外才算数。** 样本内夏普 3.0、样本外 0.2 是常态。只看 `validation` 给出的 OOS 指标。
3. **成本是策略的一部分。** 高换手策略在 A 股极易被成本吃光。回测先开成本，再谈信号。

## 四、总评

| 维度 | 评分 | 说明 |
|---|---|---|
| 方向正确性 | ★★★★★ | 把研究纪律放第一，难得 |
| 可执行性 | ★★★★☆ | 概念清晰，但缺可操作的检查动作（本框架补齐） |
| A 股适配 | ★★★★★ | 抓住 T+1/涨跌停/印花税等核心摩擦 |
| 完整性 | ★★★☆☆ | 仅阶段 1，后续待补 |

**它是一个好的起点策略。** 唯一的关键改进是：把"方法论"从知识变成**默认执行的代码动作**——这正是本框架存在的理由。
